LifeLogger API

This Swagger API describes a website called LifeLogger. With LifeLogger, you can create an account, use JWT authentication, describe habits, and track vitals like weight and other bodily stats. Users can subscribe to habits to monitor personal achievements.

Getting Started

Click on each section to expand it and see it's description and documentation.

Create an Account

To get started, use the /user/create endpoint to create a new account.

Once you've created your account and authenticated with the server (basic or JWT on the swagger, JWT only from a frontend); You may begin accessing the features, like subscribing to a habit, or tracking your vitals

Authentication

This section describes how to authenticate with the LifeLogger API using JSON Web Tokens (JWT).

To learn more about JWT, visit https://jwt.io/introduction
This website uses Simple_JWT to implement JWT authentication.

To authenticate with the LifeLogger API, you'll need to obtain a JWT from the server by sending a POST request to the /auth/token/ endpoint with your username and password in the request body. The server will respond with a JWT, which you can use to make authenticated requests to the API.

To make an authenticated request to the API, include the JWT in the header property Authorization in your HTTP request using the following format:

"Authorization": "Bearer <JWT access token>"

Use the /auth/token/refresh/ endpoint and the refresh token generated from the auth/token/ endpoint (above) to get a new access token.

Habits

This section is about tracking habits. A user can "subscribe" to a habit that they want to work on. Subscribing to a habit that doesn't exist yet will automatically create it and associate it with the user.
There are no limitations or grouping (treating upper and lowercase habits as the same habit) yet. There may also be a NSFW filter using a word set and a machine learning algorithm to try and keep the habits clean-ish.

Habits can be good or bad, with the assumption that users want to work towards good habits, and work on stopping bad habits.

Habits are tracked using the /habits/subscribe/ and /habits/unsubscribe/ endpoints.
Once a habit is subscribed to, the user can update each habit daily to help show progress (which may eventually be displayed on a graph or timeline).

Each time a user updates one of their subscribed habits, a habit history log (/habits/history /logs/list/ is generated. This endpoint can be used to generate graphs per user or per habit.

Use the /habits/subscribed/update/ endpoint to update their progress on a given habit. The habits the user is subscribed to can be found using the /habits/subscribed/list/ endpoint.

Vitals

Similar to the Habits section, this section will describe the API endpoints for tracking a user's vitals.
This section is still in active development, but will resemble the Habit section closely.
There will be a section for the user to update the vitals they're willing to give (mostly focused on weight) and an endpoint that contains the historical logs of each time a user made an update, with the ability to go back and edit or remove certain data points, in instances where they fat-fingered the number, or made a mistake.
